React Fiber

react fiber

根本原因

  • 大量的同步计算阻塞浏览器的UI渲染。既js的大量运算占用了线程。

主要几点

  1. js每运行一部分的时候把控制权交给浏览器，让浏览器有时间进行渲染，等浏览器忙完之后再继续未完成的任务。

  2. 将原来递归的方式渲染改成链表的形式进行遍历，因为链表可以暂停、继续、和丢弃。

  3. 注意一个api window.requestIdleCallback() 这个会在浏览器空闲的时候进行调用。

  4. 设置任务的优先级
